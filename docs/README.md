# 메뉴 추천

## 요구 사항

- 메뉴
  - 한 주에 같은 메뉴는 두 번 이상 먹지 않는다.
  - 같은 카테고리는 최대 2회까지만 고를 수 있다
  -> 즉, 모든 코치는 매일 다른 음식을 먹어야 하며, 같은 카테고리는 최대 2번까지 허용된다.

## 어플리케이션 흐름

1. 코치의 이름을 입력받는다.
   - "코치의 이름을 입력해 주세요. (, 로 구분)"
       - [ ] [EXCEPTION] 코치의 수가 2명보다 작거나, 5명보다 많으면 예외 발생
       - [ ] [EXCEPTION] 코치의 이름이 2글자보다 짧거나, 4글자보다 길면 예외 발생
       - [ ] [EXCEPTION] 코치의 이름이 중복되면 예외 발생
2. 각 코치가 못 먹는 메뉴를 입력받는다.
   - "~~ 이(가) 못 먹는 메뉴를 입력해 주세요." -> , 로 구분
       - 먹지 못하는 메뉴가 없으면 빈 값을 입력한다.
       - [ ] [EXCEPTION] 코치가 못먹는 메뉴가 2개보다 많으면 예외 발생
       - [ ] [EXCEPTION] 코치가 못먹는 메뉴가 중복되면 예외 발생
       - [ ] [EXCEPTION] 코치가 못먹는 메뉴가 메뉴 목록에 없으면 예외 발생
3. 메뉴를 추천한다.
   - 구분 : 요일 출력
   - 카테고리 : 한식, 일식, 중식, 양식, 아시안 중 선택된 카테고리 출력
   - 코치 이름 : 코치가 선택한 메뉴 출력
4. 프로그램을 종료한다.
   - 추천을 완료했습니다.
